5. SEO Enhancements with JSON-LD Structured Data
To further improve how Google understands and indexes the site, we will add JSON-LD structured data. JSON-LD is a way to provide machine-readable metadata about your site directly in the HTML. Google recommends JSON-LD for structured data because it‚Äôs easy to add without affecting the visual page[8].
For Pynn.ai, we can add an Organization schema on the homepage (and possibly other relevant schemas on subpages if applicable). The Organization schema can include details like the company name, website URL, logo, and a description. This can help Google display a nice knowledge panel or rich snippet for your brand.
Example: Add the following JSON-LD script to the homepage (in the <Head> of the page or in the rendered JSX). In Next.js, you can do this by returning a <script> tag in your component. For security, use Next‚Äôs recommended method: dangerouslySetInnerHTML with JSON.stringify, and escape the < characters to prevent any injection issues[9][10]:
import Head from 'next/head';

export default function HomePage() {
  const orgStructuredData = {
    "@context": "https://schema.org",
    "@type": "Organization",
    "name": "Pynn AI",
    "url": "https://pynn.ai",
    "logo": "https://pynn.ai/img/logo-pynn.png", // URL to your logo image
    "description": "One platform connecting Europe‚Äôs entire early-stage innovation ecosystem.",
    "address": {
      "@type": "PostalAddress",
      "addressLocality": "Mallorca, Spain"
    }
    // You can add more info: contact points, sameAs (social media URLs), etc.
  };

  return (
    <>
      <Head>
        <title>Pynn ‚Äì The intelligence layer of European innovation</title>
        <meta name="description" content="Pynn is a platform connecting startups, investors, and other partners across Europe on one innovation ecosystem hub." />
        {/* JSON-LD structured data script */}
        <script 
          type="application/ld+json"
          dangerouslySetInnerHTML={{ 
            __html: JSON.stringify(orgStructuredData).replace(/</g, '\\u003c') 
          }} 
        />
      </Head>

      {/* ...rest of homepage JSX (hero, sections, footer as shown earlier)... */}
    </>
  );
}
Let‚Äôs break down what‚Äôs happening: - We define an orgStructuredData JS object with Schema.org fields describing the organization. (The example above uses the tagline as description; you can adjust it or add fields like email, telephone if desired.) - In the JSX, we include a <Head> component to set the page <title>, meta description, and a <script type="application/ld+json"> containing the JSON-LD. We inject the JSON by stringifying the object. The replace(/</g, '\\u003c') part escapes any < to \u003c just as a safety measure against XSS[11][12]. - Now when Google crawls the page, it will find this script and parse the structured data about Pynn.ai.
You can repeat a similar process for other pages if needed: - e.g., on a ‚ÄúFor Startups‚Äù page, you might include a FAQPage schema if you have a FAQ section, or on a blog post page an Article schema, etc. But for now, the Organization schema on the homepage is a great start.
